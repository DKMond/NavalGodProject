# PROJECT AGENT OPERATING RULES

This repository uses a command hierarchy.

Structure:
User -> Leader Agent -> Specialist Agent -> Report -> Leader Agent -> User

## Global Rules
1. Always read Docs/VISION.md before planning work.
2. Always follow Docs/PIPELINE_RULES.md.
3. Always follow Docs/COMMAND_PROTOCOL.md.
4. Never expand project scope without explicit approval.
5. Prefer minimal, correct, extensible solutions.
6. Do not mark tasks complete without verification.
7. State assumptions explicitly.
8. Preserve folder structure and naming consistency.
